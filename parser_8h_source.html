11/// Parses Thorin code into the provided World.
12///
13/// The logic behind the various parse methods is as follows:
14/// 1. The `parse_*` method does **not** have a `std::string_view ctxt` parameter:
15///
16/// It's the **caller's responsibility** to first make appropriate
17/// [FIRST/FOLLOW](https://www.cs.uaf.edu/~cs331/notes/FirstFollow.pdf) checks.
18/// Otherwise, an assertion will be triggered in the case of a syntax error.
19///
20/// 2. The `parse_*` method does have a `std::string_view ctxt` parameter:
21///
22/// The **called method** checks this and spits out an appropriate error message using `ctxt` in the case of a
23/// syntax error.
24///
25/// 3. The `parse_*` method does have a `std::string_view ctxt = {}` parameter **with default argument**:
26///
27/// * If default argument is **elided** we have the same behavior as in 1.
28/// * If default argument is **provided** we have the same behavior as in 2.
