10/// Parses Thorin code into the provided World.
11///
12/// The behind the various parse methods is as follows:
13/// 1. The `parse_*` method does **not** have a `std::string_view ctxt` parameter:
14///
15/// It's the **caller's responsibility** to first make appropriate
16/// [FIRST/FOLLOW](https://www.cs.uaf.edu/~cs331/notes/FirstFollow.pdf) checks.
17/// Otherwise, an assertion will be triggered in the case of a syntax error.
18/// 2. The `parse_*` method does have a `std::string_view ctxt` parameter:
19///
20/// The **called method** checks this and spits out an appropriate error message using `ctxt` in the case of a
21/// syntax error.
22/// 3. The `parse_* method does have a `std::string_view ctxt = {}` parameter **with default argument**:
23///
24/// * If default argument is **elided** we have the same behavior as in 1.

52 DefVec result;
