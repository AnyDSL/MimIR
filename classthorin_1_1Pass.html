Detailed Description

All Passes that want to be registered in the PassMan must implement this interface.

  • Inherit from RWPass if your pass does not need state and a fixed-point iteration.
  • Inherit from FPPass if you do need state and a fixed-point.
  • If you do not rely on interaction between differen Passes, consider using Phase instead.

inlinevirtual

Invoked just before Pass::rewriteing PassMan::curr_nom's body.

Note
This is invoked when seeing the inside of a nominal the first time. This is often too late, as you usually want to do something when you see a nominal the first time from the outside This means that this PassMan::curr_nom has already been encountered elsewhere. Otherwise, we wouldn't have seen PassMan::curr_nom to begin with (unless it is Def::is_external).
