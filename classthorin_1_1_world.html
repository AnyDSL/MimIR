Detailed Description

The World represents the whole program and manages creation of Thorin nodes (Defs). There exists only one unique Def. These Defs are hashed into an internal HashSet (Sea/defs_). The getters just calculate a hash and lookup the Def, if it is already present, or create a new one otherwise. This corresponds to value numbering.

Use cleanup to remove dead and unreachable code.

You can create several worlds. All worlds are completely independent from each other.

Note that types are also just Defs and will be hashed as well.
