Detailed Description

template<class T>
class thorin::ArrayRef< T >

A container-like wrapper for an array.

The array may either stem from a C array, a std::vector, a std::initializer_list, an Array or another ArrayRef. ArrayRef does not own the data and, thus, does not destroy any data. Likewise, you must be carefull to not destroy data an ArrayRef is pointing to. Thorin makes use of ArrayRefs in many places. Note that you can often construct an ArrayRef inline with an initializer_list: foo(arg1, {elem1, elem2, elem3}, arg3). Useful operations are ArrayRef::skip_front and ArrayRef::skip_back to create other ArrayRefs.
