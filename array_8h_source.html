22/// A container-like wrapper for an array.
23/// The array may either stem from a C array, a `std::vector`, a `std::initializer_list`, an Array or another Span.
24/// Span does **not** own the data and, thus, does not destroy any data.
25/// Likewise, you must be carefull to not destroy data a Span is pointing to.
26/// Note that you can often construct a Span inline with an initializer_list: `foo(arg1, {elem1, elem2, elem3}, arg3)`.

28class Span {
29public:
30 using value_type = T;
31 using const_iterator = const T*;
32 using const_reverse_iterator = std::reverse_iterator<const_iterator>;

182 a.size_ = b.size_;
